# Code of Conduct

## Our Standard

We want BrickflowUI to be a project where engineers, designers, evaluators, and community contributors can collaborate without friction or intimidation.

Examples of behavior that help:

- being respectful in technical disagreement
- giving actionable feedback instead of personal criticism
- assuming good intent while still being honest about problems
- helping others understand context when a system is complex

Examples of unacceptable behavior:

- harassment, discrimination, or intimidation
- hostile or demeaning language
- publishing private information without consent
- intentionally disruptive behavior in issues, pull requests, or discussions

## Enforcement

Project maintainers may remove, edit, or reject comments, issues, commits, or pull requests that violate this code of conduct.

## Reporting

Please report conduct issues privately through the contact path described in [SECURITY.md](./SECURITY.md) or by contacting the repository owner through GitHub.

# Community And Contribution Paths

BrickflowUI is still growing, so the community surface is intentionally simple and direct.

## Best Places To Engage

- GitHub repository: [AjayAJ2000/brickflowUI](https://github.com/AjayAJ2000/brickflowUI)
- Documentation site: [ajayaj2000.github.io/brickflowUI](https://ajayaj2000.github.io/brickflowUI/)
- LinkedIn updates from the creator: [Ajay AJ](https://www.linkedin.com/in/ajayaj2000/)

## Good Ways To Contribute

- report reproducible bugs with small example apps
- improve docs clarity where the product is hard to evaluate
- suggest example apps for real-world industries and workflows
- tighten component reference quality and usage notes
- improve test coverage around input behavior, loading, media, and auth-aware patterns

## What Makes Feedback Especially Valuable

The most helpful feedback usually includes:

- what you were trying to build
- which example or component you started from
- what felt polished
- what felt limiting
- where the framework stopped feeling product-grade

That kind of context helps BrickflowUI improve as a platform, not just as a component set.
